# OpenCode-установка обязана работать в изолированной/офлайн (банк) сети

## Статус
accepted

## Контекст
После установки харнеса в проект **opencode виснет и не стартует** на изолированной корпоративной
(банк) сети — без харнеса работает. Диагностика (live, 18-07) вскрыла **три независимые причины**, все
сводятся к одному: **enforcement-плагин и install-артефакты не должны требовать сети / резолва / транспайла
на старте opencode**. `opencode --pure` обходит проблему (пропускает загрузку плагинов и snapshot) —
но это не решение, а подтверждение диагноза.

## Решение

**1. Плагин — self-contained `.mjs`, НЕ `.ts`.**
opencode грузит локальный `.ts`-плагин через **транспайл + резолв импортов + полифиллы** (`node:crypto`
→ `crypto-browserify`). В изоляции резолв/полифилл-фетч **виснет**. Сравнение подтвердило причину: **omo**
работает, т.к. это PRE-BUILT `dist/index.js` (4.9 MB, с crypto — но грузится напрямую); **старый рабочий
плагин** — 65 строк, self-contained, без crypto. Размер/crypto **не при чём** — виноват рантайм-транспайл+резолв.
→ `rational-guardrail.mjs` (plain ESM), **инлайн `shared.mjs`** (ноль внешних импортов), **cyrb53** вместо
`node:crypto` (аудит-хеши = provenance, не безопасность), без `@opencode-ai/plugin`. Дрейф инлайна ↔ `shared.mjs`
(единый источник для claude-хуков) стережёт `guardrail-sync.test.mjs`.

**2. Harness-симлинки — в `.gitignore` с ROOT-ЯКОРЕМ.**
opencode на init делает **git-snapshot** рабочей директории; git не идёт сквозь симлинк на внешний клон
(`fatal: pathspec … is beyond a symbolic link`) → **виснет**. → install дописывает в `.gitignore`:
`/.opencode/`, `/harness`, `/AGENTS.md`, `/AGENTS.harness.md`, `/.agent/`. **Ведущий `/` обязателен**: паттерн
без слэша (`harness`) матчит **вложенные** каталоги проекта (`src/harness`, `docs/AGENTS.md`) и молча
исключает их из git. _Альтернатива «копии вместо симлинков» отвергнута_ — теряет авто-раздачу dir-symlink
(обновление проекта = `git pull` клона).

**3. AGENTS.md — авто-подключение через `opencode.jsonc "instructions"`, не ручной мердж.**
Свой `AGENTS.md` проекта не трогаем; харнес-инструкции линкуем как `AGENTS.harness.md` и добавляем обе в
`instructions: ["AGENTS.md", "AGENTS.harness.md"]` → opencode грузит обе сам.

## Последствия
- **Проще:** `opencode --agent izi` стартует в изоляции без `--pure`; install-артефакты (симлинки, run-state)
  не коммитятся; dir-symlink авто-раздача сохранена; свой `AGENTS.md` не требует ручного слияния.
- **Сложнее:** плагин самодостаточен → инлайн-копия `shared.mjs` (дрейф стережёт smoke, не «просто импорт»);
  проект не может иметь **корневой** одноимённый `harness/`/`AGENTS.md` как свой (но dir-symlink install его и
  так замещает — ограничение не новое).
- **Инвариант на будущее:** любой новый install-артефакт/плагин обязан грузиться **без сети/резолва/транспайла**
  на старте opencode (профиль: pre-built или plain self-contained), иначе изоляция снова повесит старт.
