# Get started — первая задача через агентский процесс

Эта страница — пошаговое начало работы с эталоном AI-first SDLC: для **человека-дирижёра**
(Путь A) и для **ИИ-агента** (Путь B). Цель — провести одну задачу от плана до мержа,
не выдумывая процесс по ходу. Обзор и зачем — в [`README.md`](../README.md).

## Что нужно

- Репозиторий сервиса, в котором будете вести задачу (отдельный от этого concept-репо).
- Доступ к двум моделям на своей GPU-инфраструктуре: крупная (ревью/анализ) и мелкая (имплементация). Принцип асимметрии — [`GLOSSARY.md`](../GLOSSARY.md).
- Сформулированная задача в границах 1–2 модулей (agent-ready). Если шире — сначала декомпозиция[^cross] (см. [`SKILLS-BACKLOG.md`](../SKILLS-BACKLOG.md)).
- Прочитанные [`docs/00_PROCESS.md`](00_PROCESS.md) (процесс целиком) и [`AGENTS.md`](../AGENTS.md) (неизменяемые правила).

## Путь A — человек-дирижёр: провести задачу через процесс

Вы держите решения на гейтах; агенты генерируют и проверяют артефакты.

1. **Проверьте, что задача agent-ready.**
   Задача укладывается в 1–2 модуля и имеет требования. Шире границ → STOP, декомпозируйте.
   Готово, когда: есть формулировка задачи и known scope.

2. **Запустите Planner.**
   Дайте крупной модели манифест [`skills/roles/planner/planner.md`](../skills/roles/planner/planner.md) и требования задачи.
   Готово, когда: в репо сервиса появился `.agent/planner/plan.md` + контракты.

3. **Human Gate #1 — акцептуйте план.**
   Прочитайте `plan.md`: намерение, контракты, план раската. Согласны — фиксируете акцепт.
   Готово, когда: ваш аппрув записан в `.agent/decisions.log`.

4. **Запустите Implementation → CI/Review.**
   Мелкая модель по [`implementer`](../skills/roles/implementer/implementer.md) пишет код; [`fixer`](../skills/roles/fixer/fixer.md) гоняет CI и ревью.
   Готово, когда: PR зелёный, ревью без блокеров.

5. **Human Gate #2 — мерж.**
   Решение о мерже — за вами (агент не мержит).
   Готово, когда: PR влит в trunk, запись в `decisions.log`.

6. **Канареечный релиз прямо в прод (Release & Health, агент Michtom).**
   Michtom катит артефакт канарейкой за тогглом прямо в прод на вариативную среду (VM / контейнер / serverless), расширяет долю канарейки 1%→5%→25%→100% и даёт вердикт GREEN/YELLOW/RED по 4 золотым сигналам — **HG#3 приёмка прод-релиза после канарейки**.
   Готово, когда: фича стабильна на 100%, `release-health/release-health.md` = GREEN.

### Проверка пути A

Пройдите трассировку: для смерженной задачи в `.agent/decisions.log` восстановимы все
три гейта, а `plan.md` совпадает с тем, что реально выкачено. Если хоть один гейт
не зафиксирован — процесс шёл вне рамок, вернитесь к шагу.

## Путь B — ИИ-агент: войти в роль и выполнить шаг

Один агент = одна роль = один контракт вход/выход. Не импровизируй, соблюдай STOP-правила.

1. **Прочитай `AGENTS.md`.** Это неизменяемые правила разработки — твой фрейм.
2. **Загрузи манифест своей роли** `skills/roles/<role>/<role>.md`. Он перечисляет, какие скиллы из [`skills/lib/`](../skills/lib/) подключить.
3. **Проверь вход (антецедент).** Манифест задаёт входные артефакты. Их нет или неполны → **STOP**, не начинай.
4. **Выполни шаг по скиллам.** Делай по проходам скилла, соблюдай числовые лимиты и таблицы-роутеры.
5. **Запиши выход (консеквент)** в свою папку `.agent/<role>/` и решение — в `.agent/decisions.log` с фреймом (модель, скиллы@версия).

### Проверка пути B

Выходные артефакты роли существуют и полны; запись в `decisions.log` восстанавливает,
какая модель и какие версии скиллов их произвели. Иначе — шаг не считается выполненным.

## Если не сработало

**Агент начал работу без входных артефактов.**
Причина: пропущена проверка антецедента (Путь B, шаг 3).
Решение: остановить, дослать недостающий артефакт, перезапустить роль.

**Задача не лезет в 1–2 модуля, план разрастается.**
Причина: это эпик, а не стори.
Решение: декомпозировать на стори с per-service планами до запуска Planner[^cross].

**Цикл «зелёный любой ценой» — фиксы по симптому без сходимости.**
Причина: нарушена асимметрия генератор/критик или нет error-classification.
Решение: ревьюер крупнее генератора; 3 фикса по симптому → репланирование.

**Не понятно, в какой файл положить контент доки.**
Причина: не пройден роутер документации.
Решение: скилл [`documentation`](../skills/lib/documentation/SKILL.md), раздел «Роутер».
