# planning-cost-model-tiering — удешевить планирование пер-ролевым выбором модели

> Обсудить позже. Идея из наблюдения за live-прогоном.

## Проблема (замерено)
Планирование (до `mills`/Gate #1) стоит **~$1.12** / 60 вызовов, из них **50 на GLM 5.2** (все
Wirth-роли = `tier: large` = GLM). Цена драйвится GLM (дороже Qwen за токен). Не все планировочные
роли требуют силы GLM: часть — лёгкая механика/классификация.

## Идея
**Пер-ролевой выбор модели** (`models.config.json → opencode.roles`, уже поддерживается): дешёвый Qwen
для «лёгких» ролей, GLM — только для дизайна/суждения, где переусложнение/ошибка дороги.

Кандидаты (гипотеза, проверить замером):
| Роль | Нагрузка | Модель-кандидат |
|---|---|---|
| `wirth-triage` | классификация уровня (короткий вердикт) | **Qwen** (дёшево) |
| `wirth-planner` | сборка индекса путей (механика) | **Qwen** |
| `wirth-intake` | BRD→FRD (+ UC-дисциплина) | Qwen? / GLM |
| `wirth-slicer` | нарезка срезов (валидатор ловит переусложнение) | Qwen? / GLM |
| `wirth-usecase` | Cockburn use case | GLM |
| `wirth-apidesigner` | контракт (заморозка) | **GLM** |
| `wirth-moduledesigner` | дерево модулей + io + компонентные сценарии | **GLM** (самый тяжёлый) |
| `wirth-ticketer` | нарезка тикетов | GLM? / Qwen |
| `mills` | ревью-суждение | **GLM** |

## Страховка (почему риск снизился)
Детерминированные гейты (`validate-frd`/`validate-slices`/`validate-tickets` + harden-decomposition)
ловят типичные провалы слабой модели (переусложнение, битые заголовки) **механически** — значит часть
планировочных ролей можно спускать на Qwen без потери структурной корректности.

## Критерий приёмки
Найти **самую дешёвую раскладку**, при которой: (1) валидаторы зелёные, (2) декомпозиция верна (1 срез
на 1 endpoint), (3) качество артефактов не деградировало (eval-слой #42 / ручная сверка). Замерить
стоимость планирования на каждой раскладке (`usage.jsonl` cost_usd) и выбрать оптимум цена/качество.

## Механизм
Правки только в `harness/models.config.json` (`opencode.roles.<role>`), затем `gen-agents` + переустановка.
Ничего в телах ролей. Английские роли уже дают −38% токенов (см. `experiments/token-bench/role-language-token-cost.md`).

## Связи
Дополняет замер языка (role-language-token-cost) и eval-слой #42 (проверка «без потери качества» на
каждой раскладке). Артефакт: `harness/models.config.json`.
