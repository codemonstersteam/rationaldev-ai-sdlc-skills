# Темплэйт как инструмент для агента (self-describing template)

> Тикет: доработать шаблон (`template-go-api`) так, чтобы **в корне** он содержал **инструкцию для
> агента** — как запускать тесты и карту структуры проекта. Чтобы модель **не скринила весь проект**.
> Оформить шаблон **как инструмент для агента**; эту опцию использовать в тикетах.

## Проблема
Имплементер (`@hughes`) вынужден разведывать проект (глоб всех `.go`, чтение каталогов), чтобы понять
структуру и найти команды запуска тестов → жжёт токены, лезет не на свой уровень. Правило «read ONLY
ticket+inputs» это запретило — но информация должна **откуда-то браться**. Шаблон должен **сам себя
описывать**.

## Что сделать (в шаблоне `template-go-api`, корень)
Положить **корневой файл-инструкцию для агента** (напр. `PROJECT-GUIDE.md`; имя — решить, не конфликтуя
с harness-`AGENTS.md`), содержащий:

**1. Как запустить ВСЕ тесты — чёткими скриптами (команда → назначение):**
```
Выполни ЭТУ команду для юнит-тестов:        <точная команда, напр. go test ./...>
Выполни ЭТУ команду для компонентных тестов: <точная команда, напр. sh component-tests/scripts/run-tests.sh>
```
Без «пробей докер вручную» — только команда (раннер сам поднимает Docker Compose).

**2. Карта структуры проекта** — каждый раздел + **для чего его использовать при работе**:
```
cmd/app/           — точка входа (wiring), логику не держит
internal/<module>/ — модули (логика/io); где что — по module-tree
component-tests/    — харнес компонентных тестов (godog, Docker); запуск — скрипт выше
api-specification/  — замороженный контракт (OpenAPI)
docs/design/        — дизайн-пакет слайса (module-tree/contracts/c4/use-case/PLAN/tickets)
Dockerfile, docker-compose.yml — деплой сервиса (root, ≠ component-tests харнес)
config.yaml         — конфиг (порт, путь к стору)
```
Цель: агент читает **ОДИН файл-инструкцию**, а не сканирует дерево.

## Использование в тикетах
- `ticketer`: в `Verify`-строку тикета брать команды **из этой инструкции шаблона** (не хардкод в скилле);
  в `inputs` тикета указывать путь к `PROJECT-GUIDE.md`, чтобы имплементер прочитал **его**, а не глобил.
- `@hughes`/`@scaffolder`: структуру/команды берут из `PROJECT-GUIDE.md` (одно чтение), не разведывают.

## Критерий приёмки
- Шаблон в корне несёт `PROJECT-GUIDE.md`: команды unit/component + карта структуры с назначением.
- Тикеты ссылаются на него (Verify + inputs); имплементер не сканирует проект (проверить по `flow.jsonl`:
  нет `glob **/*.go` / обхода каталогов на старте тикета).

## Связи
