# patch-flow — вертикаль PATCH (`x.y.Z+1`)

> **Вес `patch`** — **обратно совместимое исправление дефекта**: код разъехался с **задокументированным
> контрактом**, правка возвращает соответствие. Контракт **не меняется**, новых модулей нет — чинится
> **секрет существующего модуля**. Транк: `Z+1`.
> Это **базовая SemVer-вертикаль**; `minor` и `major` описаны как **дельта** к ней.

## Инвариант веса (SemVer 2.0.0, дословно)

- **PATCH = backward compatible bug fix.** Решающая ось одна — **обратная совместимость документированного
  контракта**, не «баг это или поведение».
- Багфикс, который **сам** ломает совместимость (потребитель зависел от старого вывода) → **`major`**, не patch.
- Аддитивная новая способность → **`minor`**, не patch.
- **Контракт не трогается вовсе:** шаг контракта (`@wirth-apidesigner`) на patch **не выполняется**. Если
  дельта требует эволюции спеки — `@change-intake` даёт `STOP: re-triage as minor or major`.
- **Дизайн решается радиусом ряби (Parnas), а не весом** — оси ортогональны. Вопрос не «сколько модулей», а
  **«известен ли способ починки»**: `design=skip` — одна очевидная однородная правка без выбора;
  `design=needed` — правка есть **решение** (конкурирующие подходы, новая абстракция, взаимосвязь, о которой
  должны договориться несколько модулей). Радиус судит `@change-intake`.
- **Тоггла по умолчанию нет** (чинится существующее поведение, новой способности нет). Канарейка — **всегда**:
  контур релиза один, быстрой полосы для patch не бывает.

**Только native-репозитории.** Репозиторий построен самим харнесом: есть замороженный `api-specification/` и
дизайн-пакет `docs/design/slice-*/`. Разведки, conform-режима и чужих парадигм **нет** — стандарт уже в репо.
Greenfield-роли `@wirth-slicer/usecase/moduledesigner/dijkstra` и `@scaffolder` **не запускаются**.

**Прогон закрывается явно.** `.agent/` истинно только пока прогон открыт; после мерджа `@ledger` стирает
состояние, оставив запись в `docs/changes/LEDGER.md`. Иначе `gate1.approved` прошлой задачи пропустит
следующую через Gate #1: реестр даёт идемпотентность, вайп — целостность гейтов.

## Тиры

**large = суждение** (gilb, triage, change-intake, moduledesigner, ticketer, planner, mills, linger, fagan).
**medium = правка кода** (hughes-rework, michtom). **small = механика** (wirth-tester, git-hand, ledger).
Patch экономичен: тяжёлые только анализ ряби и приёмка — имплементация точечная.

## Раунды

- Dropout стадии (пусто / ошибка / нет артефакта по **точному** пути): **retry ≤2** тем же владельцем → `escalate`.
- `STOP:` — не dropout: идёт оператору, харнес останавливается; чинить/импровизировать запрещено.
- Фикс-петля `@linger`: **K=2** (гардрейл режет третью попытку) → `escalate`.
- Ревью: счётчик держит `@mills` (раунд ≥1 с блокером → `escalate`).

## Мониторинг процесса

- **Anti-loop:** гардрейл ловит субагента, повторяющего действие без прогресса → dropout → эскалация.
- **Watchdog:** плагин на `session.error` подталкивает izi продолжить с текущей точки.
- **Идемпотентность/resume:** `.agent/planner/done.log` (append-only) — izi грепает тикет перед делегированием
  и **пропускает** уже зелёный; продвижение только при маркере **И** чистом `validate-layout`.
- **Видимость:** живая строка izi на шаг + `node harness/progress.mjs .` (полоса фаз между Gate #1 и Gate #2),
  вывод **дословно** — рисовать полосу руками запрещено.
- **Человеческие гейты не прокликиваются:** #1 план, #2 мерж, #3 прод.
