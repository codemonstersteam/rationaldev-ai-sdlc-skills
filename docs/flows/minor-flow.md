# minor-flow — вертикаль MINOR (`x.Y+1.0`)

> **Вес `minor`** — **обратно совместимая новая способность**: в задокументированный контракт **добавляется**
> операция, поле или флаг, и **ни один существующий вызов не ломается**. Контракт растёт **аддитивно**;
> способность отгружается **выключенной** (тоггл default OFF). Транк: `Y+1.0`.
> Хребет общий с [`patch-flow.md`](./patch-flow.md) — ниже размечена только **Δ**.

## Инвариант веса (SemVer 2.0.0, дословно)

- **MINOR = backward compatible new functionality.** Различающий вопрос — **отсутствие → присутствие**, а не
  «старое значение → новое». Изменение, ломающее существующего потребителя, → **`major`**; починка под
  **существующий** контракт без новой способности → **`patch`**.
- **Аддитивность — ЭТО инвариант и проверяется механически:**
  `node harness/validate-contract-diff.mjs --require-additive` → **0 breaking-классов**
  (`REMOVED`, `REMOVED_OP`, `TYPE_CHANGED`, `NEW_REQUIRED`). Найден хоть один → **STOP**, вес был определён
  неверно → ре-триаж как `major`. Не «на глаз» и не суждением `@mills`.
- **Дизайн — `needed` ВСЕГДА.** Новая поверхность есть **контрактное решение** (форма, имя, место), а решение
  пиннится один раз до нарезки тикетов. Здесь `design=needed` — про **принимаемое решение**, а не про число
  модулей.
- **Способность отгружается OFF.** Тоггл default OFF: релиз этого прогона доказывает **здоровье деплоя**, а не
  работу способности под нагрузкой. **Включение — отдельный прогон оператора** со своей канарейкой и своим
  Gate #3; эта вертикаль им не владеет.
- **Существующие контрактные тесты НЕ правятся.** Правка такого теста — сигнал слома, то есть неверного веса
  (блокер `@mills`).

Только native-репозитории: замороженный `api-specification/` и `docs/design/slice-*/` уже в репо, разведки и
conform нет; `@wirth-slicer/usecase/dijkstra` и `@scaffolder` не запускаются.

## Δ-сводка — что minor добавляет к patch

1. **Семантика:** отсутствие → присутствие (новая поверхность), а не old → new существующей.
2. **Шаг контракта включается** (`@wirth-apidesigner` эволюционирует замороженную спеку аддитивно).
3. **Аддитивность механическая:** `--require-additive` = 0 breaking, иначе **STOP → `major`**.
4. **Дизайн выполняется всегда** — новая поверхность есть решение.
5. **Тоггл, default OFF**; включение — отдельный прогон.
6. **Формула тестов:** +компонентный на **новом** API, старый сьют зелёный и **нетронутый** — это и есть
   доказательство совместимости.
7. **`feat: …` / `Y+1.0`** вместо `fix: …` / `Z+1`.

## Тиры

**large = суждение** (gilb, triage, change-intake, apidesigner, moduledesigner, ticketer, planner, mills,
linger, fagan). **medium** — hughes-rework, michtom. **small** — wirth-tester, git-hand, ledger. Minor тяжелее
patch ровно на **одну всегда-включённую стадию** (контракт + дизайн добавляемой поверхности) — это цена
корректного расширения контракта; имплементация остаётся точечной.

## Раунды

- Dropout стадии: **retry ≤2** тем же владельцем → `escalate`. `STOP:` — оператору, не dropout.
- Фикс-петля `@linger`: **K=2** → `escalate`.
- Ревью: счётчик у `@mills` (раунд ≥1 с блокером → `escalate`).
- **Ре-триаж по CONTRACT_BREAKING** — не раунд, а смена веса: прогон возвращается к `@wirth-triage`.

## Мониторинг процесса

Идентичен patch ([`patch-flow.md`](./patch-flow.md) §Мониторинг): anti-loop гардрейла, watchdog на
`session.error`, идемпотентность через `.agent/planner/done.log` + `validate-layout`, живая строка izi и
`node harness/progress.mjs .` дословно. Гейты #1/#2/#3 — человеческие, не прокликиваются.
