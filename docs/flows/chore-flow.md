# chore-flow — вертикаль CHORE (no-bump)

> **Вес `chore`** — **репо-плумбинг**: CI/CD-воркфлоу, Dockerfile/compose, Makefile, конфиг линтера/форматтера,
> бамп зависимости, чистые доки. Продуктовый код и контракт **не трогаются**, чёрный ящик программы не
> меняется → **версия не двигается (`no-bump`)**. Chore — **не срез**: нет FRD, спеки, нарезки, скаффолда,
> дерева модулей и компонентных тестов. Но он **всё равно ведётся по плану и через человеческие гейты** —
> дешёвый план, а не ноль плана.

## Инвариант веса

- **Решающий тест (Ось 0 триажа, задаётся ПЕРВЫМ):** задача трогает **секрет модуля или контракт**? Добавляет
  **поведение, которое утверждал бы компонентный тест**? Если нет на оба — это `chore`.
- Правило большого пальца: артефакт — файл сборки/конфига/документации, а **чёрный ящик неизменен**.
  Спорный случай («конфиг», который на деле меняет поведение) — **не chore**, идти дальше по осям.
- **`no-bump` — нормальный исход, а не отказ.** `ci/semver-bump.mjs` классифицирует дифф по путям
  (`.github/`, `*.md`, `docs/`, `Dockerfile`, lock-файлы) → `tag=null`, тега нет, **канарейки нет**
  (`@michtom` не запускается: нечего катить на прод). Но **прогон всё равно закрывается** — запись в
  `docs/changes/LEDGER.md` и вайп `.agent/`.
- **План живёт durable, не в `.agent/`:** `docs/chores/<NNN-slug>/CHORE-PLAN.md` — своя папка = ревью-пригодно
  и переживает вайп. Гардрейл под `mode=chore` требует именно его + `gate1.approved` (а **не** полный
  `plan-review.md`).
- **Приёмка — команда верификации из плана**, а не Go-DoD `@fagan`: `validate-dod` рассчитан на срез
  сервиса/CLI, у chore среза нет.

## Тиры

**large = суждение** (gilb, triage, planner, linger). **medium** — hughes. **small** — git-hand, ledger.
Chore — самая дешёвая вертикаль: экономия достигается **вырезанием стадий**, а не ослаблением гейтов; оба
человеческих гейта остаются на месте.

## Раунды

- Dropout стадии (пусто / ошибка / нет артефакта по **точному** пути): **retry ≤2** тем же владельцем → `escalate`.
- `STOP:` — не dropout: идёт оператору, харнес останавливается.
- Фикс-петля `@linger`: **K=2** (гардрейл режет третью попытку) → `escalate`.
- Ревью `@mills` в этой вертикали нет — его роль исполняет оператор на мини-Gate #1.

## Мониторинг процесса

- **Anti-loop:** гардрейл ловит субагента без прогресса → dropout → эскалация.
- **Watchdog:** плагин на `session.error` подталкивает izi продолжить с текущей точки.
- **Идемпотентность/resume:** `.agent/planner/done.log` (append-only) — уже зелёная правка не переделывается.
- **Видимость:** живая строка izi на шаг + `node harness/progress.mjs .` дословно.
- **Человеческие гейты не прокликиваются:** мини-Gate #1 (план) и Gate #2 (мерж); Gate #3 отсутствует по
  отсутствию релиза.
