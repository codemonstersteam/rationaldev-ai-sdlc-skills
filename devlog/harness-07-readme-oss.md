# devlog/harness-07-readme-oss.md

## Задача

Подготовить проект к open-source: вынести харнес в основной README (быстрый старт с
Claude) и вычистить из документации внутренние/сравнительные тексты, не нужные
читателю-консьюмеру. PR #10 и #11.

## Контекст и ограничения

- README ревьюился рубрикой `doc-quality-review` (6 характеристик IBM).
  Главные ✗: task orientation (старт «что делать» был внизу/в блок-цитате),
  organization (порядок не «лестницей»), clarity/completeness (нет «что получишь»).
- OSS-читателю нужно «что есть / как подключить», а не «что мы убрали vs big-tech».

## Принятое решение

**README (PR #10):**
- Секция «Быстрый старт (Claude Code)» вверху: `git clone` → `./install.sh claude <proj>`
  → как работать (3 шага) → что получишь. Агенты/имена izi — оставлены ниже.
- Харнес добавлен в «Навигацию» (`docs/story-harness/`, `harness/`).
- Терминология: «дерзкий стартап / стартаперский» → «дерзкая разработка с ИИ»;
  «стартаперский CD» → «дерзкий CD».

## Результат

- README task-ориентирован; вхождений `стартап`/`big-tech`/мета-сравнений нет.
- `./harness/smoke/run.sh` → PASS 20 на всём протяжении.
- Проект готов к рассылке в сообщество (архив-снимок репозитория).
