# token-bench — эксперимент: стоимость разработки харнесом vs без

Контролируемый A/B: сколько **денег, токенов и времени** уходит на одну и ту же задачу
у разных способов «разработки с ИИ» — и какого качества выходит результат. Метрика честности —
**один арбитр** (`acceptance/check.sh`) и **одна замороженная задача** (`spec/task.md`).

> Это воспроизводимый стенд. Полная инструкция для запуска (оператором или агентом) — [`RUNBOOK.md`](RUNBOOK.md).
> Секреты (ключ провайдера) в гит НЕ хранятся — только `proxy/.env.example`.

## Армы (на одной задаче и одном арбитре)

| Арм | Раннер | Установка |
|---|---|---|
| **Голый Claude** | Claude Code, без харнеса | — (baseline) |
| **Харнес на Claude** | Claude Code + харнес | `./install.sh claude <sb>` |
| **Харнес на OpenCode** | OpenCode + харнес | `./install.sh opencode <sb> --hard` |
| **omo** | OpenCode + oh-my-openagent | глобальная установка omo |

## Метод
- **Одна задача** — `spec/task.md` (frozen): модульный Go-сервис, OpenAPI-first, конфиг из файла
  (без хардкода), Gherkin/godog компонентные тесты в Docker, unit-тесты, README + карта отказов.
- **Одна палитра моделей** по тирам: `opus-4-8 / sonnet-4-6 / haiku-4-5`.
- **Замер — на gateway:** Go-прокси `proxy/` форвардит на провайдера и логирует per-request
  `input_tokens` (вкл. `cache_read`/`cache_creation`) + `output_tokens` в JSONL. Каждый арм пишет
  в **свой** файл (логи не затирать).
- **Арбитр** `acceptance/check.sh`: build + `go test` + endpoint==эталон + модульность + OpenAPI +
  Gherkin-в-Docker + README. Что не соответствует — не принимается.
- Агенты недетерминированы → гонять N раз, сравнивать распределения.

## Конфигурация песочницы
Чистый каталог: `go.mod` (`module bench-task`) + `services.yaml` (фикстура-стор) + `TASK.md`
(копия спеки в cwd, иначе агент ищет снаружи). Под арм ставится харнес/omo (или ничего для
baseline). Модели вшиваются по тирам. Подробности и команды — `RUNBOOK.md`.

## Результаты — test0 (OpenCode-армы; Claude-армы предстоят)

| Арм | check.sh | output-токены | $ (incl. input+кэш) | время | прерывания |
|---|---|---|---|---|---|
| **Харнес (OpenCode)** | ✅ PASS | 123 352 | **≈ $2.3** | ≈ 18–20 мин | 0 |
| **omo** | ✅ PASS | 123 761 | **$10.37** | ≈ 28 мин активных | 2× кредит-лимит |
| Голый Claude | _предстоит_ | | | | |
| Харнес (Claude) | _предстоит_ | | | | |

**Качество (test0):** оба арма выдали полную поставку по спеке (модули `internal/{storage,catalog,
httpapi}`, OpenAPI, config, godog-в-Docker, unit, README). Харнес — компактнее (336 строк кода
vs 532), больше гранулярных unit-функций (17 vs 13), и оставил **дизайн-пакет** (FRD/C4/contracts/
slices). omo — многословнее, артефакты эквивалентны.

## Находки
- **Output-токены почти равны (~123K), но omo дороже в ~4.5×.** Разница — в **input/контексте**:
  omo впрыскивает огромный контекст (Sisyphus-ultraworker, окно ~191K, чтение своих референсов),
  input-токены раздувают цену при том же выходе. → Меряй стоимость по **total $ (input+output+кэш)
  на завершённую задачу**, не по output-токенам.
- **Баг enforcement-плагина** (резолв корня в `/` → EROFS + ложная блокировка `@implementer` под
  headless `opencode run`) найден и исправлен — см. **PR #26**. Прокси доработан под честный учёт
  input (вкл. кэш-токены).
