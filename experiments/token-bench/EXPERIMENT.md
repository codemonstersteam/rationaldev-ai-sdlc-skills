# Эксперимент: стоимость разработки — харнес vs omo vs голый Claude

Контролируемый A/B: сколько денег, токенов и времени уходит на одну и ту же задачу у разных
способов разработки с ИИ, и какого качества результат. Один арбитр (`acceptance/check.sh`),
одна замороженная задача (`spec/task.md`). Воспроизведение — `RUNBOOK.md` (⚠️ нужен `tmux`).

## Армы
| Арм | установка |
|---|---|
| **Голый Claude** | Claude Code, без харнеса (baseline) |
| **Харнес на Claude** | `./install.sh claude <sb>` |
| **Харнес на OpenCode** | `./install.sh opencode <sb> --hard` |
| **omo** | глобальный oh-my-openagent (Ultimate, SUL-1.0) |

## Метод
Чистая песочница (`go.mod` + `services.yaml`-стор + `TASK.md`), под арм ставится харнес/omo,
модели по тирам `opus/sonnet/haiku`. Замер — Go-прокси `proxy/` (форвард на провайдера +
per-request лог `input_tokens` вкл. `cache_read`/`cache_creation` и `output_tokens`). Каждый арм —
свой лог (не затирать). Прогон наблюдаемый через **tmux**. Недетерминизм → N прогонов.

## Результат — test0 (финальная спека: модули + OpenAPI + godog-в-Docker + unit + README)

| Арм | check.sh | output-токены | $ (incl. input+кэш) | время | прерывания |
|---|---|---|---|---|---|
| **Харнес (OpenCode)** | ✅ PASS | 123 352 | **≈ $2.3** | ≈ 18–20 мин | 0 |
| **omo** | ✅ PASS | 123 761 | **$10.37** | ≈ 28 мин активных | 2× кредит-лимит |
| Голый Claude | _предстоит_ | | | | |
| Харнес (Claude) | _предстоит_ | | | | |

**Качество:** оба арма выдали полную поставку (модули `internal/{storage,catalog,httpapi}`, OpenAPI,
config-из-файла, godog-в-Docker, unit, README + карта отказов). Харнес — компактнее (336 строк
кода vs 532), больше unit-функций (17 vs 13), оставил **дизайн-пакет** (FRD/C4/contracts/slices).

## Ключевая находка
**Output-токены почти равны (~123K), но omo дороже в ~4.5×.** Вся разница — в **input/контексте**:
omo впрыскивает огромный контекст (Sisyphus-ultraworker, окно ~191K, чтение своих референсов).

➡️ **Меряй стоимость по total $ (input+output+кэш) на завершённую задачу, не по output-токенам.**
Также: кэш-токены, рост контекста на шаг, cost-per-success, success-rate, микс моделей по тирам,
предохранители (лимиты токенов/шагов), N прогонов.
