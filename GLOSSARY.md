# Глоссарий

Один канонический термин на понятие; синонимы — в `_Avoid_`.

## Вес и вертикали

**Вес (weight)**:
SemVer-класс задачи (`greenfield · patch · minor · major · chore`), задающий глубину дизайна, набор
тестов, префикс ветки/PR и бамп транка. Решающая ось — **обратная совместимость документированного
контракта**. Определяет `@wirth-triage`; `izi` роутит по вердикту механически.
_Avoid_: размер, сложность, объём задачи.

**Вертикаль**:
Конвейер одного веса от BRD до тега на транке. Пять вертикалей делят общий хребет (Gate #1 →
реализация → DoD → PR/CI → Gate #2 → закрытие прогона) и отличаются головой.
_Avoid_: маршрут, режим, ветка процесса.

**Аддитивность**:
Свойство эволюции контракта на `minor`: ни один существующий элемент не удалён, не переименован, не
сменил тип и не стал обязательным. Проверяется механически —
`node harness/validate-contract-diff.mjs --require-additive` → 0 breaking-классов; найден слом ⇒ вес
был неверен ⇒ ре-триаж как `major`.
_Avoid_: «обратно совместимо на глаз».

**Discriminating-сценарий**:
Проверка, различающая старое и новое поведение **на данных** (для `minor` — отсутствие → присутствие).
Изменение без такой проверки не доказано; если детерминированно запиннить нельзя, причина названа в DoD.
_Avoid_: «регрессия не сломалась».

**Тоггл (feature toggle)**:
Ключ, которым новая способность отгружается **выключенной** (default OFF) и открывается канарейкой по
доле трафика; он же — ручка мгновенного отката без передеплоя. Включение способности — отдельный
прогон со своим Gate #3.
_Avoid_: флаг конфигурации.

## Закрытие прогона

**Пруф мерджа**:
Подтверждение из forge, что PR **влит** (`state == MERGED`) — предусловие тега.
_Avoid_: «PR зелёный» (зелёный CI ≠ влито).

**no-bump**:
Исход закрытия, при котором версия не двигается: дифф трогает только плумбинг (`.github/`, `*.md`,
`docs/`, `Dockerfile`, lock-файлы). Тега нет, канарейки нет — **нормальный** исход (типичный для
`chore`), а не отказ.
_Avoid_: «пустой релиз», «пропуск версии».

**Закрытие прогона (run-close)**:
Перевод сделанной работы из **состояния** (`.agent/`) в **запись** (`docs/changes/LEDGER.md`) с
очисткой состояния: пруф мерджа → тег → запись → атомарный вайп. Выполняет только `@ledger` через
`harness/close-run.mjs`. Без него `gate1.approved` прошлой задачи пропустил бы следующую через гейт.
_Avoid_: «финализация», «постмерж-хук».

**Форма тега**:
Наличие/отсутствие префикса `v` — берётся у последнего релизного тега репозитория; если тегов нет,
действует дефолт `v` (`DEFAULT_PREFIX` в `ci/semver-bump.mjs`) — ровно один раз.

## Роль = модуль

**Роль работает как программный модуль** с контрактом вход/выход:

- **Вход — проверка полноты данных.** Роль не стартует, пока её входные артефакты не существуют и не
  полны (антецедент). Невыполнение → STOP.
- **Выход — контроль выходных данных.** Роль отвечает за полноту и корректность того, что пишет
  (консеквент).

Поэтому «разложить по модулям» = «разложить по ролям»: каждая роль — контракт
`skills/roles/<role>/<role>.md`. Он **генерируется** из единого источника
`harness/agents/_shared/<role>.md` — руками не правится.

**Асимметрия генератор/критик**:
Кто генерирует артефакт (план, код, выкат), тот его не принимает; оценщик не мельче генератора.
Отсюда: `@fagan` принимает, но не чинит; `@linger` чинит, но не принимает; гейты держит человек.
