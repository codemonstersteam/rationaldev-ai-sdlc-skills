# Глоссарий

## Роль = модуль

**Роль работает как программный модуль** с контрактом вход/выход:

- **Вход — проверка диапазона значений (полноты данных).** Роль не стартует, пока
  её входные артефакты не существуют и не полны (антецедент). Невыполнение → STOP.
- **Выход — контроль выходных данных.** Роль отвечает за полноту и корректность
  того, что пишет в свою папку `.agent/<роль>/` (консеквент).

Поэтому «разложить по модулям» = «разложить по ролям»: каждая роль — контракт
`skills/roles/<role>/<role>.md` (что грузить, вход/выход, STOP). Он **генерируется** из
единого источника `harness/agents/_shared/<role>.md` — руками не правится.

## Соответствия терминов

| Термин в intent / обиходе | Канон (по процессу) |
|---|---|
| модуль | агентская роль (§3.2 процесса) |
| проектировщик фичи | **Planner** на уровне сервиса |
| доменный скилл / «ролевая обёртка» | `roles/<role>/skills/<domain>.md` (architecture, code-style, …) |
| техника | `roles/<role>/skills/<technique>/` (program-design, http-io, …) |
| план фичи (источник правды) | `.agent/planner/plan.md` в репо сервиса |
| `backlog.md` | рудимент; не источник правды |

## Роли (модули) и модели

| Роль | Агент (izi) | Модель | Шаг процесса |
|---|---|---|---|
| Planner | **Wirth** | крупная | PLANNING |
| Plan Reviewer | **Mills** | крупная | PLAN REVIEW |
| Implementer | **Hughes** | мелкая | IMPLEMENTATION |
| Fixer / Code Reviewer | **Linger** | крупная (ревью) / мелкая (фикс) | CI + CODE REVIEW |
| Release & Health | **Michtom** | мелкая (выкат) / крупная (анализ) | RELEASE (канарейка + 4 сигнала) |
| Engineer-Orchestrator | **Witt** (человек) | человек | Входной gate, Human Gate #1–3 |

> Набор агентов — **izi**. Бывшие роли Release + SRE слиты в одну «Release & Health».

**Асимметрия генератор/критик:** тот, кто генерирует артефакт (план, код, выкат),
не тот, кто его принимает. Ревьюер/оценщик — всегда крупнее генератора.
