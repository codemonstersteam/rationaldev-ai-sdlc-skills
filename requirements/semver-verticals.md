# semver-verticals — FRD

## Problem statement

Харнес умеет вести задачу только «с нуля» (greenfield) и разнородными rework-маршрутами, а
пост-мерж-контур отсутствует: конвейер кончается на PR. Нужно, чтобы **вес работы задавал SemVer**
(greenfield · patch · minor · major · chore) и чтобы транк автоматически получал версию-тег после
акцептованного человеком мерджа.

## Actors & external systems

| Актор / система | Роль | Интерфейс (порт) |
|---|---|---|
| **Оператор** (человек-дирижёр) | ставит задачу, держит Gate #1/#2/#3 | CLI-сессия агента `izi`; токены `GATE1 APPROVE` / `GATE2 APPROVE` |
| **Целевой репозиторий** | где выполняется работа | файловая система + `git` (транк, ветка `<type>/<slug>`, теги) |
| **Forge** (GitHub `gh` / Bitbucket REST) | PR, статус CI, список тегов, факт мерджа | `gh` CLI · REST; выбор — `.agent/planner/vcs`, реестр `harness/vcs-providers.json` |
| **CI целевого репо** | зелёный/красный статус PR | статус-чек PR (читается через forge) |
| **Плагин-гардрейл харнеса** | ставит маркеры гейтов, блокирует имплементеров | хуки сессии, `.agent/gates/*.approved` |

Внешних сетевых интерфейсов харнес не добавляет: тег ставится через `git`/forge, версия читается из
тегов транка (**VERSION-файла нет** — нечему разъезжаться).

## Use cases

### UC-1 — Роутинг задачи по весу (SemVer)

- **Primary actor:** оператор. **Stakeholders:** все роли конвейера.
- **Precondition:** есть измеримый BRD (`.agent/planner/brd.md`, выдан `@gilb`).
- **Success guarantee:** записан `.agent/planner/mode` с одним значением
  `greenfield | patch | minor | major | chore`; `izi` запустил соответствующий лейн.

**MSS**
1. `@wirth-triage` читает BRD и существующий репозиторий.
2. Применяет решающий тест — **одна ось: обратная совместимость документированного контракта**
   (SemVer 2.0.0, дословно): ломает совместимость → `major`; иначе добавляет функциональность →
   `minor`; иначе совместимый багфикс → `patch`.
3. Пустой репозиторий / v0→v1 → `greenfield`; изменение только плумбинга (CI, доки, Dockerfile,
   lock-файлы), не трогающее продуктовый код и контракт → `chore`.
4. Возвращает строку вердикта с `route=<вес>` и пишет маркер `mode`.
5. `izi` роутит по **фиксированной таблице** (механика, не суждение).

**Extensions**
- 2a. Багфикс, который сам ломает совместимость → **`major`**, не `patch` (решает совместимость, а не
  «это баг») → `WEIGHT_MISCLASSIFIED` предотвращён на входе.
- 3a. Задача многорепная (epic) → **STOP**, алгоритм не реализован, сообщить оператору.
- 4a. Вес не определяется из BRD (нет контракта, чтобы судить о совместимости) → **STOP**, открытый
  вопрос оператору; выдумывать вес запрещено.

### UC-2 — Проведение задачи веса `patch`

- **Primary actor:** оператор. **Precondition:** `mode=patch`, репозиторий построен харнесом
  (есть design-пакет и `api-specification/`).
- **Success guarantee:** код сошёлся с задокументированным контрактом; контракт **не изменился**;
  весь сьют зелёный; PR открыт заголовком `fix: …`.

**MSS**
1. `@change-intake` → `<change-dir>/change-delta.md`: затронутые модули, **discriminating-сценарий**
   old ≠ new на данных, `design=needed|skip` по радиусу ряби (Parnas).
2. При `design=needed` — дизайн затронутых модулей (`module-tree`/`contracts`/`c4` + `adr/`).
3. `@wirth-ticketer` → тикеты; покрытие идёт **туда, где различие наблюдаемо**: доходит до эндпоинта →
   `component`-тикет (модуль `blocked_by` его, RED-first); остаётся внутри модуля → discriminating unit.
4. `@wirth-planner` → `PLAN.md`; `@mills` → вердикт; **Gate #1**.
5. `@git-hand mode=start` → ветка `fix/<slug>` от свежего транка → имплементация → `@fagan`.
6. `@git-hand mode=terminal` → commit → push → PR (`fix: …`) → ограниченное ожидание CI.

**Extensions**
- 1a. Различие не пиннится никаким детерминированным тестом (гонка, латентный путь) → отгружается без
  discriminating-теста, **причина названа в DoD**, доказательство — зелёный существующий сьют.
- 3a. Существующий component-тест утверждал старое (дефектное) значение → он **исправляется** в том же
  тикете (восстановление инварианта, а не ослабление ассерта).
- 6a. `ci=red` → `@linger` (предохранитель K=2) → повтор terminal; `ci=pending-timeout` — это ответ, а
  не зависание.

### UC-3 — Проведение задачи веса `minor`

- **Success guarantee:** контракт вырос **аддитивно** (ни один существующий элемент не удалён, не
  переименован, не сменил тип, не стал обязательным); новая способность отгружена **выключенной**.

**MSS**
1. `@change-intake` → спецификация НОВОЙ поверхности (операция/поле/флаг), сценарий
   **отсутствие → присутствие**, утверждение обратной совместимости; `design=needed` **всегда**.
2. `@wirth-apidesigner` эволюционирует `api-specification/*` аддитивно; `validate-contract-diff.mjs
   --require-additive` → **0 breaking-классов**.
3. Тикеты: (a) `component` на НОВОЕ API (RED-first: до — 404/отсутствует/дефолт), (b) модульные тикеты,
   (c) требование тоггла, **default OFF**.
4. Далее как UC-2, ветка `feat/<slug>`, PR `feat: …`.

**Extensions**
- 2a. Диффер нашёл breaking-класс → **STOP**, ре-триаж как `major` (вес был определён неверно).
- 3a. Тикет правит существующий контрактный тест → блокер `@mills`: правка существующего теста —
  сигнал слома, то есть неверного веса.

### UC-4 — Проведение задачи веса `major`

- **Success guarantee:** несовместимое изменение проведено с миграционным путём и явным
  breaking-списком в теле PR; формула тестов держится (`N = 1 + Σ ветвей`), изменённые компонентные
  тесты доработаны под новый контракт.
- **MSS:** как UC-3, но дизайн — редизайн + миграция/депрекейшн; PR несёт `BREAKING CHANGE`.
- **Extensions:** 1a. Миграционный путь для существующих потребителей не определяется из BRD → **STOP**,
  открытый вопрос оператору.

### UC-5 — Закрытие прогона: тег SemVer на транке (пост-Gate #2)

- **Primary actor:** оператор (акцептует мерж), исполнитель — `@ledger`.
- **Precondition:** PR зелёный, оператор выдал `GATE2 APPROVE`, гардрейл поставил
  `.agent/gates/gate2.approved`, ветка **влита** в транк.
- **Success guarantee:** на транке стоит тег следующей версии; в `docs/changes/LEDGER.md` есть
  самодостаточная запись; `.agent/` состояние прогона атомарно стёрто.

**MSS**
1. `@ledger` вызывает детерминированный `harness/close-run.mjs --pr <N>` (сам ничего не вычисляет).
2. Скрипт читает вес из `.agent/planner/mode`, факт мерджа и файлы PR из forge, список тегов —
   из `git ls-remote origin` (не из локальных тегов, они отстают).
3. `ci/semver-bump.mjs` решает следующий тег: `patch → Z+1`, `minor → Y+1.0`, `major → X+1.0.0`,
   `greenfield → 1.0.0`; **форма тега берётся у последнего релизного тега репо** (`1.2.3` или `v1.2.3`).
4. Запись в `LEDGER.md` (append-only) — **до** вайпа; затем атомарный вайп `.agent/`.
5. `@ledger` возвращает одну строку результата.

**Extensions**
- 1a. Маркера Gate #2 нет → скрипт отказывает; роль **не может** создать маркер (самоакцепт запрещён).
- 1b. Ветка ещё не влита → отказ: пруф мерджа обязателен, тег ставится только на влитый транк.
- 3a. Дифф — только плумбинг (`.github/`, `*.md`, `docs/`, `Dockerfile`, lock-файлы) → **no-bump**:
  тега нет, канарейки нет; это **нормальный** исход, не отказ (`chore` — обычный его источник).
- 3b. Целевой тег уже существует → отказ без перезаписи (`TAG_EXISTS`).
- 4a. Запись в `LEDGER.md` не удалась → вайп **не выполняется** (порядок причинный: вайп стирает то,
  что читают первые два акта).

## Data dictionary

| Поле | Тип / домен | Обяз. | `error.code` при нарушении |
|---|---|---|---|
| `.agent/planner/mode` | enum `greenfield\|patch\|minor\|major\|chore` | да | `MODE_INVALID` |
| `route=` в строке триажа | тот же enum | да | `ROUTE_UNPARSEABLE` |
| ветка | `^(feat\|fix\|refactor\|chore)/[a-z0-9-]+$`, срез от свежего транка | да | `BRANCH_INVALID` |
| заголовок PR | Conventional Commits, префикс выведен из `mode` (`patch→fix:`, `minor→feat:`, `major→feat!:`/`BREAKING CHANGE`, `chore→chore:`) | да | `PR_TITLE_MISMATCH` |
| тег | `^v?\d+\.\d+\.\d+(-[0-9A-Za-z.-]+)?(\+[0-9A-Za-z.-]+)?$`; форма (`v`/без) — от последнего релизного тега репо | нет (no-bump) | `TAG_MALFORMED` |
| `.agent/gates/gate2.approved` | маркер-файл, ставит **только** гардрейл по токену `GATE2 APPROVE` | да для UC-5 | `GATE2_MISSING` |
| факт мерджа PR | булево из forge (`state == MERGED`) | да для UC-5 | `PR_NOT_MERGED` |
| `--require-additive` (minor) | число breaking-классов = 0 | да для minor | `CONTRACT_BREAKING` |
| тоггл новой способности | значение по умолчанию **OFF** | да для minor | `TOGGLE_DEFAULT_ON` |
| classification плумбинга | путь ∈ {`.github/`, `*.md`, `docs/`, `Dockerfile`, lock-файлы} → no-bump | — | — |

## Glossary

Термины закреплены в корневом [`GLOSSARY.md`](../GLOSSARY.md); специфичные для этой работы:

- **Вес (weight)** — SemVer-класс задачи, задающий глубину дизайна, набор тестов, префикс ветки/PR и
  бамп транка. _Avoid:_ «размер», «сложность» — они не решают совместимость.
- **Вертикаль** — конвейер одного веса (greenfield/patch/minor/major/chore) от BRD до тега.
- **Пруф мерджа** — подтверждение из forge, что PR влит; предусловие тега. _Avoid:_ «PR зелёный» —
  зелёный CI ≠ влито.
- **no-bump** — исход, при котором версия не двигается: дифф не трогает продуктовый код и контракт.
- **Закрытие прогона** — перевод сделанной работы из **состояния** (`.agent/`) в **запись**
  (`LEDGER.md`) с очисткой состояния, чтобы `gate1.approved` прошлой задачи не пропустил следующую.

## Contract (draft)

Внешнего сетевого API нет — контракт харнеса это **CLI + артефакты**:

```
node ci/semver-bump.mjs --weight <patch|minor|major|greenfield> --files-from <f> [--tags-from <f>] [--json]
  → { "tag": "v1.3.0" | null, "reason": "bump|no-bump", "form": "v"|"" }
node harness/close-run.mjs --pr <N> [--json]
  → tag → append docs/changes/LEDGER.md → wipe .agent/   (порядок фиксирован)
```

Артефакты-обязательства: `.agent/planner/{brd,mode,change-dir}`, `<change-dir>/{change-delta,PLAN}.md`,
`<change-dir>/tickets/ticket-N.md`, `.agent/gates/gate{1,2}.approved`, `docs/changes/LEDGER.md`.

## Failure-mode map

| `error.code` | Когда | Действие оператора |
|---|---|---|
| `ROUTE_UNPARSEABLE` / `MODE_INVALID` | триаж не вернул вес | переформулировать BRD, назвать контракт |
| `WEIGHT_MISCLASSIFIED` | minor-дифф оказался breaking | принять ре-триаж как `major` |
| `CONTRACT_BREAKING` | `--require-additive` нашёл слом | STOP → `major` |
| `TOGGLE_DEFAULT_ON` | новая способность включена по умолчанию | вернуть тикет на доработку |
| `GATE2_MISSING` | закрытие прогона до акцепта мерджа | выдать `GATE2 APPROVE` |
| `PR_NOT_MERGED` | тег до влития | смержить PR, повторить закрытие |
| `TAG_EXISTS` / `TAG_MALFORMED` | конфликт/битая форма тега | разобрать теги репо вручную |
| `LEDGER_WRITE_FAILED` | запись не легла | чинить и повторять; `.agent/` **не** стёрт |

## NFR / constraints

1. **Строгий стандарт, только native-репозитории.** Харнес ведёт репозитории, построенные им самим;
   conform-режим и разведка чужих парадигм удалены (см. Ш1) — стандарт навязывается, а не подстраивается.
2. **Тег — механика, не суждение.** Роль вызывает скрипт и зеркалит его строку; собственной арифметики
   версий у ролей нет.
3. **Человеческие гейты неавтокликабельны** (#1 план, #2 мерж, #3 прод после канарейки); маркеры ставит
   только гардрейл по точному токену.
4. **Ядро `semver-bump` чистое (`io: none`)** → тестируется без git/CI; побочки — в CLI-хвосте.
5. Роль/скилл ≤ 300 строк; формулировки плотные, якорными понятиями.
6. Источник правды версии — **тег транка**; VERSION-файла не заводим.
