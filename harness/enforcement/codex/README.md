# Codex enforcement — почему остаётся на инструкции

`--hard` для Codex **не даёт жёстких гарантий**: у Codex бедная хук-поверхность
(`notify`/exec, без блокирующих pre-tool хуков уровня OpenCode-плагина или Claude
`PreToolUse`). Заблокировать делегирование роли или гарантированно дописать
`decisions.log` механически — нечем.

## Что делаем вместо enforcement

- Гейты и `decisions.log` остаются **инструкцией** в `AGENTS.md` (роутер + блоки ролей),
  как в базовом режиме — единообразно с Claude/OpenCode без `--hard`.
- Маркер апрува `.agent/gates/gate1.approved` и запись в `.agent/decisions.log` —
  на дисциплине модели и оператора.

## Когда это приемлемо

- Сильная модель надёжно следует инструкции (см. «База vs `--hard`» в гайде).
- Если нужен жёсткий аудит/гейт — используй раннер с enforcement (OpenCode-плагин
  или Claude-хуки), а Codex — для прогонов, где хватает инструкции.

> Если в будущем у Codex появятся блокирующие pre-tool хуки — адаптер ляжет сюда
> по тому же контракту: блок `implementer` без апрува + append `decisions.log`.
