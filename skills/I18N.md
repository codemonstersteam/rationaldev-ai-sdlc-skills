# i18n скиллов — глоссарий и литералы (Фаза 0)

Опора для перевода `skills/lib/*` на английский (см. план в `backlog.md`). Цель —
единый перевод терминов во всех скиллах и защита кросс-скилл строк от рассинхрона.

## Раскладка тиров (если скилл называет модель роли)

`large` = **opus** (planner / plan-reviewer / fixer / orchestrator) · `medium` = **sonnet**
(release-health) · `small` = **haiku** (implementer). Тир — абстракция; конкретную модель
несёт манифест роли, скилл MUST ссылаться на тир, не пинить модель.

## Глоссарий RU → EN (единый перевод)

| RU | EN | RU | EN |
|---|---|---|---|
| срез / слайс | slice | контракт | contract |
| головной модуль / голова | head module / head | антецедент / консеквент | antecedent / consequent |
| ингресс-адаптер | ingress adapter | предусловие / постусловие | precondition / postcondition |
| труба | pipe | режим отказа | failure mode |
| развилка (по полю) | branch | карта режимов отказа | failure-mode map |
| развилка (на дороге) | decision point | доменная структура / сущность | domain struct / entity |
| дерево модулей | module tree | конструктор | constructor |
| сквозная инфраструктура | cross-cutting infrastructure | хендофф | handoff |
| канарейка | canary | выкат / раскатка | rollout |
| фиче-тоггл / тоггл | feature toggle (flag) | стенд / тестовая среда | test environment |
| гейт / человеческий гейт | gate / human gate | золотые сигналы | golden signals |
| ревьюер / критик | reviewer / critic | асимметрия генератор↔критик | generator/critic asymmetry |
| юнит / компонентный / контрактный тест | unit / component / contract test | покрытие | coverage |
| спека | spec | роутер-таблица | router table |
| STOP-правило | STOP rule | чеклист | checklist |
| наблюдаемость | observability | бюджет | budget |
| пошаговое уточнение | stepwise refinement | минимизация / минимализм | minimalism |

> «Слабая модель» MUST переводиться как тир (`small`/`medium`/`large`), а не «weak model
> (Qwen3.5)» — tier-agnostic (раннер маппит тир на свою модель).

## Литералы — строки, которые ОДИН скилл пишет, а ДРУГОЙ матчит

Эти строки MUST оставаться **по-русски**, пока и писатель, и читатель не переведены
(иначе сломается межскилловый матч). Переводятся в Фазе 3, всей цепочкой сразу.

| Литерал | Пишет | Матчит / читает |
|---|---|---|
| `## Карта режимов отказа` (README) | architecture / security (в plan) | program-design (Шаг 0), component-tests |
| `## Хендофф-чеклист` | program-design (Шаг 12) | program-implementation (Шаг 0) |
| `- [x] Оператор аппрувит пакет — @<handle>, <YYYY-MM-DD>` | program-design | program-implementation (Шаг 0) |
| `## Решения по дизайну` (карточка слайса) | program-design | program-implementation (Шаг 0) |
| `## Архитектура изменения` (plan.md) | architecture | plan-reviewer |
| `## Границы изменения` (plan.md) | architecture | plan-reviewer |
| `## Безопасность` (plan.md) | security | plan-reviewer |
| `## Наблюдаемость и проверка работоспособности` (plan.md) | observability | plan-reviewer, release-health |
| Поля девлога: `## S<n>`, `**Что сделано:**`, `**Решения по ходу:**`, `**Что застряло:**`, `**Тесты:**` | program-implementation | оператор (человек) |

**НЕ литералы** (язык-нейтральны — переводить/оставлять свободно): `## Gherkin-mapping`
(уже EN), пути `.agent/...`, имена `error.code` / sentinel, маркеры `decisions.log`
(`role=`, `via=`), машинные спеки (OpenAPI/AsyncAPI).
