# Markdown Formatting & Self-Check Skill

## Назначение

Скилл для проверки и исправления форматирования Markdown-документов перед коммитом.
На вход — `.md` файл (или несколько файлов).
На выход — отформатированный файл без ошибок форматирования.

**Цель:** Все разделы, списки и заголовки отображаются корректно в MkDocs/Material for MkDocs.

---

## Шаг 0. Проверка обязательных артефактов

**Вход:** `.md` файл. **Выход:** решение «форматируем / стоп».

**STOP-правила:**

- Файл не в кодировке UTF-8 → остановиться, сконвертировать
- Файл содержит бинарные данные → остановиться, сообщить оператору
- Файл не существует → остановиться, проверить путь

---

## Шаг 1. Проверка заголовков

### 1.1. Заголовки `##` и `###`

**Правило:** После каждого заголовка уровня `##` или `###` должна быть **пустая строка**.

**Проверка:**

```bash
# Найти заголовки без пустой строки после
grep -n '^#{2,3} .\+$' file.md | while read line; do
  linenum=$(echo "$line" | cut -d: -f1)
  nextlinenum=$((linenum + 1))
  nextline=$(sed -n "${nextlinenum}p" file.md)
  if [ -n "$nextline" ]; then
    echo "⚠️  Строка $linenum: Заголовок без пустой строки после"
  fi
done
```

**Исправление:**

```bash
# Автоматически добавить пустую строку после заголовков ###
python3 << 'EOF'
import re

with open('file.md', 'r', encoding='utf-8') as f:
    content = f.read()

# Добавляем пустую строку после заголовков ### если её нет
content = re.sub(r'^(### .+)$', r'\1\n', content, flags=re.MULTILINE)
content = re.sub(r'^(## .+)$', r'\1\n', content, flags=re.MULTILINE)

with open('file.md', 'w', encoding='utf-8') as f:
    f.write(content)
EOF
```

### 1.2. Жирные заголовки разделов

**Правило:** После жирных заголовков вида `**Текст:**` должна быть **пустая строка** перед списком.

**Примеры:**

```markdown
✅ Правильно:
**См. также:**

- [Ссылка 1](file1.md)
- [Ссылка 2](file2.md)

**Чего станет МЕНЬШЕ:**

- Пункт 1
- Пункт 2

❌ Неправильно:
**См. также:**
- [Ссылка 1](file1.md)
- [Ссылка 2](file2.md)

**Чего станет МЕНЬШЕ:**
- Пункт 1
- Пункт 2
```

**Проверка:**

```bash
# Найти жирные заголовки без пустой строки
grep -n '^\*\*.*:\*\*$' file.md | while read line; do
  linenum=$(echo "$line" | cut -d: -f1)
  nextlinenum=$((linenum + 1))
  nextline=$(sed -n "${nextlinenum}p" file.md)
  if [ "$nextline" != "" ] && [[ ! "$nextline" =~ ^- ]]; then
    echo "⚠️  Строка $linenum: Жирный заголовок без пустой строки"
  fi
done
```

**Чеклист жирных заголовков:**

- [ ] `**См. также:**` — пустая строка после
- [ ] `**Соответствие уровням:**` — пустая строка после
- [ ] `**Треки после Senior:**` — пустая строка после
- [ ] `**Чего станет МЕНЬШЕ:**` — пустая строка после
- [ ] `**Чего станет БОЛЬШЕ:**` — пустая строка после
- [ ] `**Для кого этот профиль:**` — пустая строка после (если используется как заголовок)
- [ ] `**Назначение уровня:**` — пустая строка после (если используется как заголовок)

## Шаг 2. Проверка списков

### 2.1. Списки после заголовков

**Правило:** Между заголовком и списком должна быть **пустая строка**.

**Примеры:**

```markdown
✅ Правильно:
### Задачи

- Задача 1
- Задача 2

❌ Неправильно:
### Задачи
- Задача 1
- Задача 2
```

**Проверка:**

```bash
# Python-скрипт для проверки
python3 << 'EOF'
import re

with open('file.md', 'r', encoding='utf-8') as f:
    lines = f.readlines()

issues = []
for i, line in enumerate(lines):
    # Заголовок уровня ###
    if re.match(r'^#{3}\s+.+', line):
        if i + 1 < len(lines):
            next_line = lines[i + 1]
            if next_line.strip() != '' and not next_line.strip().startswith('-'):
                issues.append(f"Строка {i+1}: Заголовок ### без пустой строки")
    
    # Жирный заголовок
    if re.match(r'^\*\*.+\*\*$', line.strip()):
        if i + 1 < len(lines):
            next_line = lines[i + 1]
            if next_line.strip() != '' and not next_line.strip().startswith('-'):
                issues.append(f"Строка {i+1}: Жирный заголовок без пустой строки")

if issues:
    for issue in issues:
        print(f"⚠️  {issue}")
else:
    print("✅ Списки: OK")
EOF
```

### 2.2. Нумерованные списки

**Правило:** Нумерованные списки должны иметь пустую строку до и после.

**Пример:**

```markdown
✅ Правильно:
## Раздел

1. Пункт 1
2. Пункт 2

Текст после.

❌ Неправильно:
## Раздел
1. Пункт 1
2. Пункт 2
Текст после.
```

## Шаг 3. Проверка таблиц

### 3.1. Формат таблиц

**Правило:** Таблицы должны иметь пустую строку до и после.

**Пример:**

```markdown
✅ Правильно:
## Таблица

| Колонка 1 | Колонка 2 |
|-----------|-----------|
| Ячейка 1  | Ячейка 2  |

Текст после.

❌ Неправильно:
## Таблица
| Колонка 1 | Колонка 2 |
|-----------|-----------|
| Ячейка 1  | Ячейка 2  |
Текст после.
```

**Проверка:**

```bash
# Найти таблицы без пустой строки до/после
grep -n '^|.*|$' file.md | head -1
```

### 3.2. Выравнивание колонок

**Правило:** Разделительная строка должна иметь дефисы для каждой колонки.

```markdown
✅ Правильно:
| Left | Center | Right |
|:-----|:------:|------:|

❌ Неправильно:
| Left | Center | Right |
|------|--------|-------|
```

## Шаг 4. Проверка ссылок

### 4.1. Относительные ссылки

**Правило:** Ссылки на другие `.md` файлы должны существовать.

**Проверка:**

```bash
# Найти все ссылки и проверить существование файлов
grep -oE '\[.*\]\([^)]+\)' file.md | while read link; do
  url=$(echo "$link" | grep -oE '\([^)]+\)' | tr -d '()')
  if [[ "$url" =~ \.md$ ]] && [[ ! "$url" =~ ^http ]]; then
    if [ ! -f "$(dirname file.md)/$url" ]; then
      echo "⚠️  Ссылка не существует: $url"
    fi
  fi
done
```

### 4.2. Якоря (anchors)

**Правило:** Якоря на разделы в том же файле должны соответствовать заголовкам.

**Пример:**

```markdown
✅ Правильно:
[См. раздел](#заголовок-раздела)

## Заголовок раздела

❌ Неправильно:
[См. раздел](#несуществующий-якорь)
```

## Шаг 5. Проверка блоков кода

### 5.1. Ограждение блоков кода

**Правило:** Блоки кода должны иметь пустую строку до и после.

**Пример:**

```markdown
✅ Правильно:
Текст перед.

```bash
echo "Hello"
```

Текст после.

❌ Неправильно:
Текст перед.
```bash
echo "Hello"
```
Текст после.
```

### 5.2. Указание языка

**Правило:** Всегда указывать язык для блоков кода.

```markdown
✅ Правильно:
```python
def hello():
    print("Hello")
```

❌ Неправильно:
```
def hello():
    print("Hello")
```
```

## Шаг 6. Финальный чеклист

Перед коммитом проверить:

### Заголовки
- [ ] После всех `##` и `###` есть пустая строка
- [ ] После `**Текст:**` (жирные заголовки) есть пустая строка
- [ ] Нет заголовков без текста после (кроме последних в файле)

### Списки
- [ ] Все списки имеют пустую строку перед началом
- [ ] Нумерованные списки имеют пустую строку после конца
- [ ] Нет смешения нумерованных и ненумерованных списков без причины

### Таблицы
- [ ] Все таблицы имеют пустую строку до и после
- [ ] Разделительная строка имеет правильное количество колонок
- [ ] Нет таблиц без заголовков

### Ссылки
- [ ] Все относительные ссылки на `.md` файлы существуют
- [ ] Все якоря соответствуют заголовкам
- [ ] Нет битых ссылок (проверить `mkdocs build`)

### Блоки кода
- [ ] Все блоки кода имеют пустую строку до и после
- [ ] Указан язык для каждого блока кода
- [ ] Нет блоков кода без ограждения

### Общее
- [ ] Файл в кодировке UTF-8
- [ ] Нет trailing whitespace в конце строк
- [ ] Файл заканчивается одной пустой строкой (не двумя)

## Автоматизация проверки

### Pre-commit hook

Создать `.git/hooks/pre-commit`:

```bash
#!/bin/bash

# Проверяем только .md файлы
md_files=$(git diff --cached --name-only --diff-filter=ACM | grep '\.md$')

if [ -z "$md_files" ]; then
    exit 0
fi

echo "Проверка Markdown форматирования..."

for file in $md_files; do
    echo "Проверка: $file"
    
    # Проверка заголовков
    python3 << 'EOF'
import re
import sys

with open('$file', 'r', encoding='utf-8') as f:
    lines = f.readlines()

issues = []
for i, line in enumerate(lines):
    if re.match(r'^#{2,3}\s+.+', line):
        if i + 1 < len(lines) and lines[i + 1].strip() != '':
            issues.append(f"Строка {i+1}: Заголовок без пустой строки")

if issues:
    for issue in issues:
        print(f"❌ {issue}")
    sys.exit(1)
EOF
    
    if [ $? -ne 0 ]; then
        echo "❌ Проверка не пройдена: $file"
        exit 1
    fi
done

echo "✅ Все проверки пройдены"
exit 0
```

### CI проверка

Добавить в `.github/workflows/ci.yml`:

```yaml
jobs:
  markdown-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Check Markdown formatting

          pip install pymdown-extensions

## Быстрые команды для проверки

```bash
# 1. Найти все заголовки без пустой строки
grep -B1 '^#{2,3} ' docs/**/*.md | grep -v '^--$'

# 2. Найти жирные заголовки без пустой строки
grep -B1 '^\*\*.*:\*\*$' docs/**/*.md | grep -v '^--$'

# 3. Проверка сборки MkDocs
mkdocs build

# 4. Автоматическое исправление
python3 scripts/fix-md-formatting.py docs/
```

## STOP-правила

Остановиться и сообщить оператору, если:

- Файл содержит более 1000 строк и требует ручного ревью
- Обнаружено более 20 ошибок форматирования в одном файле
- Ссылки ведут на несуществующие файлы в других репозиториях
- Таблицы имеют более 10 колонок (возможно требуется рефакторинг)
