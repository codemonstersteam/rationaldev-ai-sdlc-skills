# Роль: Engineer-Orchestrator (Инженер-дирижёр)

- **Имя (izi, honorary):** **Witt** — со-автор *Structured Programming: Theory **and**
  Practice*. Держит границу «теория ↔ практика» на Human Gates. Единственная роль-человек.
- **Версия:** 1.0
- **Исполнитель:** человек (не агент)
- **Шаг процесса:** Входной gate, Human Gate #1–3, разбор эскалаций (§9)
- **Принцип «роль = модуль»:** держит контроль на границах — на намерении (план)
  и результате (ревью + CI), а не построчно на коде.

## Грузить (из `skills/lib/`, пути `../../lib/`)

### Техники
- `doc-quality-review/` — оценка качества плана/доки на Gate #1.
- `platform-landing/` — координация кросс-сервисной фичи/эпика.
> **ПРОБЕЛ (Блок 2):** завести
- `agent-ready-gate/` — входной чеклист «задача в границах 1–2 модулей» (§10).
- `error-classification/` — разбор эскалаций: дефект плана / реализации / конфигурации / ложная тревога.

## Точки принятия решений (вход → решение)
- **Входной gate.** Вход: требования. Задача готова и agent-ready? → запуск Delivery / переформулировать.
- **Human Gate #1 — акцепт плана.** Вход: `plan.md` + `plan-review.md`. → принять / на доработку / вмешаться.
- **Разбор эскалаций (Ralph Loop).** Вход: `.agent/memory.md` (что пробовали и почему застряли, skill `memory/`) + `decisions.log` (история решений). → перезапуск с правками / переписать план / стоп.
- **Human Gate #2 — мерж.** Вход: финальный PR + вердикт code review. → мерж (триггерит канареечный выкат в прод).
- **Разбор аномалий раскатки.** Вход: RED от Release & Health (Michtom). → в фикс / перепланировать / ложная тревога.
- **Human Gate #3 — приёмка прод-релиза.** Вход: стабильный GREEN после канарейки (доля до 100%) + ручная проверка непокрытого. Значимые фичи — подтверждает; мелкие — авто (fast-path).

## Выход
- Решения на gate'ах → `.agent/decisions.log` (трассируемость, инженерная ценность).

## Компетенции (§9)
Архитектура и проектирование (главное), чтение кода на незнакомых стеках, ясная постановка задачи, управление группой агентов, многозадачность с фокусом.
